Requirements - International Dashboard

We received requirements for the International Dashboard from Adrian’s product team. They provided a low fidelity wireframe (below) which we based our mocks off of. The requirements have been molded since the initial project kickoff.

  • Dashboard of Order IDs

    A dashboard with rows for each order. Metadata columns that need to be included are..

    1. 1. Order ID

    2. 2. SLA Date

    3. 3. Go Live Date

    4. 4. Channel

    5. 5. Media Type

    6. 6. Status


    Users can sort the table in ascending or descending order for any column.

  • Pipeline

    Each order ID displays the linear chain of progress states; filled in ones express that it is on or has finished that state. The states need to be grouped accordingly from its source. The states (statuses) are...

    1. 1. Order Placed (Lacuna)

    2. 2. Order Creation Required (Pipeline)

    3. 3. Creating Order (Pipeline)

    4. 4. Order Referral (Pipeline)

    5. 5. Ready at Platform (Playout)

    6. 6. Status


    The status column will display where in the pipeline the order currently is located. Hovering over the latest state allows users to see the breakdown of the overall state i.e. progress of Video, Engligh Audio, Spanish Audio, etc.

  • Search

    Users need the ability to search the dashboard for specific Order IDs so they can find specific ones quickly.

  • Filters

    Ability to filter the dashboard by….

    1. 1. Go live date (On and In-Between)

    2. 2. SLA Date (On and In-Between)

    3. 3. Channel

    4. 4. Media

    5. 5. Status


    Users need the ability to save and manage custom filter settings as well for shortcuts.

  • Notifications

    The dashboard needs to notify users if orders are waiting, as risk, or complete. Clicking on one of those should filter the dashboard accordingly.
