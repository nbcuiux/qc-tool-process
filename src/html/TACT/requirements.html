Requirements - TACT Dashboard/QC Tool

We received requirements for the QC Tool from Adrian’s product team. They provided a low fidelity wireframe (below) which we based our mocks off of. The requirements have been molded since the initial project kickoff. Users access the QC tool for a specific order through the TACT dashboard.

Contents

  • TACT Dashboard
  • QC Tool

TACT Dashboard

  1. Dashboard of order IDs

    1. A dashboard with rows for each order. Metadata columns that need to be included will reflect the international dashboard. We are still waiting on confirmation from product team.

    2. Users can sort each column by ascending or descending order

    3. A column for error log so users can access the error log for each order

  2. Ability to search the dashboard by order ID

    1. Users need the ability to search the dashboard for specific Order IDs so they can find specific ones quickly

  3. Ability to filter the dashboard by….

    1. Go live date (On and In-Between)

    2. SLA Date (On and In-Between)

    3. Channel

    4. Media

    5. Status

  4. Users need the ability to save and manage custom filter settings as well for shortcuts

  5. Notifications

    1. The dashboard needs to notify users if orders are waiting, as risk, or complete. Clicking on one of those should filter the dashboard accordingly

  6. Ability to navigate through multiple pages of Order IDs

  7. Error Log Modal

    1. The modal will display a grid of submissions that includes columns for XML, Mezzaine, High, Medium, and Low to tell the user if it passed or failed. Each submission will have a Creative ID & a Creative Ad

    2. Users need the ability to check the most recent submission at the top

    3. Ability for users to resubmit a failed one

QC Tool

  1. Metadata of the order in the header

  2. Ability to assign the order ID a state

    1. Pass

    2. Fail

    3. Escalate

  3. Ability to view player proxy of video

    1. Ability to identify a start and end time which is mapped to the start & end times in the new comment areas

  4. Ability to add new comments

    1. Identify comment type

    2. Comment timecode (Is mapped to player proxy timecode)

  5. Feed of Comments Panel

    1. Shown in chronological order

    2. Could display comments from Baton

    3. Ability to edit comments already added

  6. Ability to go back to the TACT dashboard
